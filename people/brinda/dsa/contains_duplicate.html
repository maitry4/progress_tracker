Contains Duplicate

View on LeetCode →

Problem Statement

Given an integer array nums, return true if any value appears at least twice in the array, and return false if every element is distinct.

Approaches

Approach 1: Brute Force (Not Recommended)

Time Complexity: O(n²)

Space Complexity: O(1)

Check every element against every other element. This is inefficient for large arrays.

Approach 2: Sorting

Time Complexity: O(n log n)

Space Complexity: O(1)

Sort the array and check if adjacent elements are equal.

C++ Implementation:

class Solution {
public:
    bool containsDuplicate(vector<int>& nums) {
        sort(nums.begin(), nums.end());
        for (int i = 1; i < nums.size(); i++) {
            if (nums[i] == nums[i - 1]) {
                return true;
            }
        }
        return false;
    }
};

Java Implementation:

class Solution {
    public boolean containsDuplicate(int[] nums) {
        Arrays.sort(nums);
        for (int i = 1; i < nums.length; i++) {
            if (nums[i] == nums[i - 1]) {
                return true;
            }
        }
        return false;
    }
}

Approach 3: Hash Set (Optimal)

Time Complexity: O(n)

Space Complexity: O(n)

Use a hash set to track seen elements. If we encounter an element already in the set, we have a duplicate.

C++ Implementation:

class Solution {
public:
    bool containsDuplicate(vector<int>& nums) {
        unordered_set<int> seen;
        for (int num : nums) {
            if (seen.count(num) > 0) {
                return true;
            }
            seen.insert(num);
        }
        return false;
    }
};

Java Implementation:

class Solution {
    public boolean containsDuplicate(int[] nums) {
        HashSet<Integer> seen = new HashSet<>();
        for (int num : nums) {
            if (seen.contains(num)) {
                return true;
            }
            seen.add(num);
        }
        return false;
    }
}

Key Insights

  • Hash sets provide O(1) average-case lookup time
  • Trading space for time is often worthwhile
  • The optimal solution depends on constraints (memory vs time)
